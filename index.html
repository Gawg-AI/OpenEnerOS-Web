【 一 】 为什么能源需要"原生内核"？

当通用大模型遇到电力系统，会暴露出五大根本性痛点。EnerOS 的诞生，正是为了逐一破解它们。

◆ 痛点一：通用模型不懂电网

传统大模型对电力系统是"外挂式"理解 —— 它能背出潮流方程，却不懂为何一条线路跳闸会让半个省的负荷在 200 毫秒内转移。能源的复杂性，藏在物理约束与拓扑耦合里，无法靠提示词补齐。
◆ 痛点二：物理约束即真理

基尔霍夫定律、潮流边界、继电保护约束是硬性的物理事实。我们把这些写进内核，Agent 的每一次决策都要先过物理可行性这道"闸" —— 它不是事后校验，而是决策本身的 前置条件。
◆ 痛点三：从"问答"到"行动"

能源场景里，Agent 必须能真正下达指令：投切电容器、调整机组出力、隔离故障区段。EnerOS 为智能体提供一套 原生的执行接口，让模型从"会回答"进化到"会操作"。
◆ 痛点四：毫秒级实时性

电力系统的扰动以毫秒计，通用 IT 系统的秒级响应远远不够。EnerOS 内核采用 双执行架构，把实时控制与通用认知分离，确保关键决策在确定的时间窗口内完成。
◆ 痛点五：海量时序数据

电网每时每刻都在产生量测、事件、状态数据。传统"先存储再查询"的模式跟不上节奏。EnerOS 把 时序作为一等公民，让 Agent 直接在数据流的"现场"做判断。
◆ EnerOS 的回答

面对这五大痛点，EnerOS 给出的答案是：让 物理网格成为第一公民，让 智能体成为电网节点，让 约束成为内核法则。这是一次从"应用智能"到"原生智能"的范式跃迁。

【 二 】 双执行架构 + 电力原生内核

EnerOS 不是一层封装，而是从最底层的电网物理模型，一路向上搭到智能体决策层。最具特色的是它的 双执行域 设计：通用执行域负责认知与编排，实时执行域负责确定性的物理控制，二者各司其职、互不阻塞。

第 四 层 智能体编排层 多智能体协同、任务分解、记忆与反思。调度、运维、规划、交易、故障自愈、能效六类原生 Agent 在此运行，可被业务系统直接调用。智能体之间通过事件总线协作，像一支训练有素的调度班组。
第 三 层 决策安全层 把行业规程、运行约束、调度边界编码为策略。任何动作在执行前都要过"安全闸" —— 违规动作被拦截，并给出 可解释理由。策略即代码，权限分级，全程留痕。
第 二 层 认知模型层 融合电力领域语料与物理机理训练的认知模型，理解设备语义、拓扑结构、量测信号与时序。让 Agent 懂"电网在说什么"，而不只是"文字在说什么"。
第 一 层 电力原生内核 物理网格是第一公民 —— 节点、支路、潮流、保护。这是 EnerOS 与所有通用 OS 的根本区别。双执行架构：通用执行域处理认知编排，实时执行域保障物理控制的确定性。

【 三 】 十二大内核 Crate —— 器官级的电力运行时

EnerOS 的内核由 12 个 Rust Crate 组成。它们各司其职，从拓扑、潮流到设备、网关， 像一套精密协作的"器官"，共同支撑起完整的电力智能体运行时。全部以内存安全、零成本抽象的 Rust 编写。

◆ eneros-core
整个系统的根基。定义内核抽象、执行域调度、模块加载与生命周期管理。所有其他 Crate 都构建在它之上。		 ◆ topology
把变电站、线路、开关、变压器组织为可计算的图结构。支持动态合环/解环，Agent 看到的永远是"此刻真实电网"。		 ◆ powerflow
内核的"数学引擎"。牛顿-拉夫逊与快速解耦法并存，毫秒级给出电压、功率、损耗分布，是物理可行性的裁判。
◆ constraint
基尔霍夫定律、容量上限、电压越限、N-1 校验。把不可违背的物理事实编码为内核法则，决策的硬边界。		 ◆ equipment
为每个设备赋予类型、参数、状态与行为。让 Agent 理解"这台开关能做什么、不能做什么"，而非一串 ID。		 ◆ timeseries
量测、事件、状态以时间序列原生流转。无需先落库再查询，Agent 直接在数据流"现场"做判断与预测。
◆ eventbus
连接所有 Crate 与 Agent 的神经。拓扑变化、量测越限、设备动作都以事件实时广播，支持发布订阅与优先级。		 ◆ gateway
对接 IEC 61850、Modbus、MQTT、DNP3 等工业协议。让现场设备的数据与控制指令无缝进出内核。		 ◆ device
统一不同厂家、不同型号设备的接口差异。上层 Agent 只需面对一致的抽象，无需关心底层协议细节。
◆ api
面向业务系统的标准 API 层。REST、gRPC、WebSocket 一应俱全，让 EnerOS 能被 ERP、SCADA、营销系统调用。		 ◆ bridge
在异构系统间翻译数据与语义。打通老旧自动化系统与现代智能体，让历史投资继续发光。		 ◆ network
内核节点间的可靠通信层。支持高可用集群与边缘部署，断网时本地 Agent 依然能守住物理安全底线。

【 四 】 六大原生智能体 —— 电网的"细胞"

EnerOS 内置六类原生智能体，覆盖电力系统核心业务。它们通过事件总线协同， 智能体即电网节点（Agent-as-Grid-Node），不是旁观者，而是电网自身的延伸。

[ Agent 01 ]
调度 Agent

面对高比例新能源的随机性与波动，在物理内核上滚动求解最优潮流。它把"经验调度"升级为"可解释的最优调度"，每一次调整都经过潮流校验，把人类调度员从高强度重复决策中解放。
[ Agent 02 ]
运维 Agent

融合视频、红外、局放、振动等多模态信号，理解设备状态语义。它能自主生成缺陷报告、联动工单、安排检修窗口，把无人值守变电站从愿景变成日常。
[ Agent 03 ]
规划 Agent

基于长期时序与拓扑演化趋势，给出网架优化、电源接入、负荷预测方案。它考虑的不仅是当下可行，更是未来五到十年的弹性与经济性。
[ Agent 04 ]
交易 Agent

在物理可行边界内聚合分布式资源，参与现货与辅助服务市场。每一笔报价都经过潮流校验，杜绝"纸面可行、物理违约"，让虚拟电厂真正可调度。
[ Agent 05 ]
故障自愈 Agent

故障发生的瞬间，基于实时拓扑自主完成隔离、转供、恢复的闭环决策。把传统分钟级的应急压缩到秒级，最大限度减少停电范围与时长。
[ Agent 06 ]
能效 Agent

协调"源-网-荷-储"，在物理可行下持续寻优。它让每一度电都流向最该去的地方，挖掘降损空间，把能效优化从"定期评估"变成"实时经营"。

【 五 】 六大设计哲学 —— 刻进内核的信仰

★ Power-Native First
电力优先

电力不是 EnerOS 的一个"模块"，而是它的本体。所有能力都围绕电网物理世界构建，智能体从诞生起就"住"在电网里，而非外挂其上。
★ Agent-as-Grid-Node
智能体即电网节点

智能体不是电网之外的观察者，而是电网拓扑中的一个节点。它能感知、能决策、能行动，与设备、线路、保护共同构成电网的生命体。
★ Constraint as Kernel Law
约束即内核法则

物理约束不是事后校验的清单，而是写进内核的不可违背的法则。任何决策都要先满足物理可行性，违规动作在源头就被禁止。
★ Time-Series Native
时序原生

时间序列数据不是被存储后查询的"档案"，而是实时流动的"血液"。Agent 直接在数据流的现场工作，对动态变化作出即时响应。
★ Real-Time Determinism
实时确定性

关键决策必须在确定的时间窗口内完成。双执行架构把实时控制与通用认知分离，确保物理控制永远及时、可预测、不抖动。
★ Open & Interoperable
开放互操作

EnerOS 不造孤岛。标准协议、开放接口、可插拔架构，让它能与现有自动化系统、设备、平台无缝协作，保护每一分历史投资。
